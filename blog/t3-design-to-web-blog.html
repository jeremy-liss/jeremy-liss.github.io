Responsive sites

A responisve site displays differntly depending on the kind of device that is being used to access it. You can have a different design which is optomised to look best for each device, and to work better depending on whether the user will be ineracting with a mouse or swiping.

Mobile first design

Mobile first design means beginning with the design most optimised for mobile and then making exceptions for larger devices. This is because sites for larger devices usually have more data and larger images to load - desiging mobile first means that mobiles won't need to load up all this information which they would if it was designed desktop first.

Frameworks

A framework is a template of CSS, HTML or js files which can be used to standardise code and solve common problems such as browser compatiability and responsivness. They provide a common structure so that it's not neccesary for developers to write it all out themselves.

    Pros
  • Standardised code
  • Neat and tidy code
  • Automatic browser compatiability
  • Responsive
    Cons
  • Less control
  • Unused code
  • Less learning through doing it yourself

Wireframes

A wireframe is a simple sketch of the layout of a website, drawn up in the early stages of the design process. It's a bit like a blueprint for a house.
